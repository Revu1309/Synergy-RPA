DATA FLOW EXAMPLE: Crypto Update Cycle
─────────────────────────────────────────────────────────────────────────────

Time: 10:00:00
  ↓
  └─→ Scheduler Job Starts: scrape_and_store()
      ↓
      └─→ Fetch from Binance: [15 cryptos]
          ↓
          └─→ Data Service normalizes each
              ├─→ BTC: {'symbol': 'BTC', 'price_usd': 45000, 'timestamp': '10:00:05Z'}
              ├─→ ETH: {'symbol': 'ETH', 'price_usd': 2500, 'timestamp': '10:00:05Z'}
              └─→ ... 13 more
                  ↓
                  └─→ Validate each entry
                      ↓
                      └─→ Insert to database
                          ├─→ INSERT INTO crypto_assets (15 records)
                          ├─→ INSERT INTO crypto_latest (15 UPSERT)
                          └─→ ✓ Database updated
                              ↓
                              └─→ Update Cache
                                  ├─→ Compute hash of new data
                                  ├─→ Compare with old hash
                                  ├─→ Hashes differ → NOTIFY
                                  └─→ ✓ Cache updated
                                      ↓
                                      └─→ Notify Subscribers
                                          ├─→ Browser connected to SSE
                                          ├─→ Server sends event
                                          ├─→ Browser receives live update
                                          └─→ ✓ UI updates in real-time
                                              ↓
                                              └─→ No refresh needed!

Time: 10:00:30
  Same data (no change)
      ↓
      └─→ Compute hash = SAME as before
          ├─→ Hash matches → SKIP notification
          └─→ ✓ Redundant update prevented

Time: 10:05:00
  New data available
      ↓
      └─→ Compute hash ≠ DIFFERENT from before
          ├─→ Hash changed → NOTIFY
          └─→ Cycle repeats...

  ✓ 50-75% fewer database queries (cache reduces load)
  ✓ 90% faster dashboard loads (cached data)
